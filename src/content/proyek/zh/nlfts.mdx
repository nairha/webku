### 什么是 NLFTs？

**NLFTs** 是一个印度尼西亚开发者社区，专注于**开发者体验（Dx）**、**工具与库**的开发，以及开发者之间高水准的技术讨论。该社区被设计为一个密集协作空间，**24 小时不间断**运行，专为印度尼西亚地区的开发者而设。

NLFTs 不仅仅是一个聊天群组——它是一个拥有**自身标准**、**自身许可证**和**自身治理体系**（称为**君主制**）的开发者生态系统。

## 愿景与使命

### 愿景

> **建立一个现代、独立且具有全球竞争力的印度尼西亚开发者社区——专注于发展适应现代标准的本地技术生态系统。**

NLFTs 展望一个印度尼西亚，开发者不仅是外国技术的**消费者**，更是其他开发者所用工具、库和平台的**创造者**——无论是在印度尼西亚还是在全球。

### 使命

1. **发展本地 Dx 生态系统**——构建诞生于印度尼西亚、为印度尼西亚服务并准备走向全球的开发者工具和库。
2. **在印度尼西亚提升 Nuxt.js**——使 Nuxt 成为旗舰框架，并改变 Nuxt 在印度尼西亚开发者生态中被“轻视”的看法。
3. **制定我们自己的标准**——通过 FTs Standard 和 FTs License，NLFTs 为内部生态系统设定具体的质量和规则。
4. **创造 24/7 高质量讨论**——提供一个不眠的技术讨论空间，每个问题和意见都受到重视。
5. **通过技术会议进行教育**——每月举行会议，评估新技术：哪些值得采用，哪些应该放弃。
6. **建立技术独立性**——减少对低效旧技术的依赖，推动现代技术的采用。
7. **保持排他性而不 elitism**——对所有满足基本条件的开发者开放，不论背景。

## 加入途径

#### 1. Discord（开放）

- **访问权限：** 所有符合资格的开发者
- **限制：** 必须年满 17 岁（符合 Discord 政策）
- **排他性级别：** **低**——最广泛的入口
- **加入方式：** 邀请链接 → 年龄验证 → 进入

#### 2. WhatsApp 群组（有条件开放）

- **访问权限：** 所有符合资格的开发者
- **限制：** 除一般资格外无年龄限制
- **排他性级别：** **中**——有清理机制
- **加入方式：** 推荐链接 → 资格验证 → 进入
- **警告：** **不活跃 1-2 个月**的成员将被**永久**移除，且**无法重新加入**

#### 3. Discord 贡献者（专属）

- **访问权限：** 仅通过基于实际贡献的筛选
- **限制：** 必须在 NLFTs 有贡献记录
- **排他性级别：** **高**——最有价值的空间
- **进入标准：**
  - 积极提交 Pull Request
  - 积极修复问题
  - 参与高价值讨论
  - 获得核心团队或领导者的认可
- **福利：**
  - 访问专属频道
  - 在技术讨论中拥有更大话语权
  - 在活动和项目中获得优先权
  - 在 NLFTs 生态中拥有最高声誉

### 活跃规则

| 渠道 | 不活跃时限 | 后果 |
|---------|------------------|-------------|
| Discord | 无严格限制 | 无自动移除 |
| WhatsApp | **1-2 个月** | **永久移除，无法返回** |
| Discord 贡献者 | 定期评估 | 若贡献停止则撤销状态 |

### 成员移除

成员可能因以下原因被移出 NLFTs：

1. **不活跃**（仅限 WhatsApp——1-2 个月无活动）
2. **违反行为准则**（参见[行为准则](#行为准则)章节）
3. **违反适用政策**
4. **注册时伪造资格**
5. **对 NLFTs 造成机构性损害的行为**

> 因 WhatsApp 不活跃而被移除是**自动且永久的**。没有申诉流程。

## NLFTs 标准与许可证

NLFTs 有两个内部监管支柱，使其区别于任何其他社区：

### FTs Standard

**FTs Standard** 是 NLFTs 为以下方面设定的质量标准：

- **代码：** 编写风格、结构、命名约定、架构
- **工具与库：** 被视为 NLFTs 生态系统一部分的最低质量要求
- **文档：** 格式、深度、语言
- **贡献：** 审查流程、PR、问题跟踪
- **活动：** 组织标准、材料、演讲者质量

FTs Standard 是**不断演进**的——根据技术发展通过每月会议更新。

### FTs License

**FTs License** 是应用于 NLFTs 生态系统所有产出的许可证，包括：

- 在 NLFTs 下开发的工具与库
- 官方文档
- 书面标准与政策
- 活动和研讨会材料

> FTs Standard 和 FTs License 的完整细节见单独文件。

## 技术重点

### 主要优先事项

NLFTs 目前有 **3 个主要技术支柱**，作为开发和推广的重点：

#### 1. Nuxt.js（重点 #1）

**为什么选择 Nuxt？**

- 在印度尼西亚，Nuxt.js 相比于 Next.js 常常被**轻视**
- NLFTs 决心在印度尼西亚**建立 Nuxt 的声誉**
- Nuxt 拥有出色的 Dx，符合 NLFTs 对开发者体验的关注
- 愿景：使 Nuxt 成为印度尼西亚开发者进行严肃项目的**首选**

**NLFTs 为 Nuxt 所做的工作：**
- 深入讨论 Nuxt 架构
- 开发 Nuxt 模块和插件
- 通过会议和活动进行教育
- 为全球 Nuxt 生态做出贡献

#### 2. TypeScript + Vue.js（重点 #2）

- TypeScript 作为主要语言标准
- Vue.js 作为首选 UI 框架
- TS + Vue 组合为现代应用开发提供最佳 Dx
- 推动全面采用 TypeScript（严格模式）

#### 3. Rust（重点 #3）

- Rust 作为工具、CLI 和性能关键组件的语言
- 符合 NLFTs 对 Dx 工具的关注
- 在印度尼西亚建立稳固的 Rust 社区
- 连接 Rust 与 JS/TS 生态（WASM、NAPI 等）

### 支持的技术

除了 3 个主要优先项，NLFTs 还支持和讨论：

| 类别 | 技术 |
|----------|--------------|
| **后端** | NestJS、Hono、Bun、Deno、Fastify、tRPC |
| **前端** | Vue 3、React（作为替代）、Svelte、Solid |
| **移动端** | Flutter、React Native、Tauri（桌面） |
| **数据库** | PostgreSQL、Redis、MongoDB、Drizzle ORM、Prisma |
| **DevOps** | Docker、Kubernetes、GitHub Actions、Vercel、Cloudflare |
| **语言** | TypeScript、JavaScript、Rust、Go、Python、Kotlin |
| **工具** | Vite、Turbo、Nx、Biome、pnpm |

### 不支持的技术

NLFTs 明确**不讨论、不支持、不鼓励**使用被认为**低效**或**传统**的技术：

| 不支持 | 原因 |
|---------------|--------|
| **PHP 8 及以下** | 旧技术，Dx 低，不符合现代愿景 |
| **Ruby** | 不符合 NLFTs 生态重点 |
| **jQuery** | 对现代项目已过时 |
| **无预处理器/框架的纯 CSS** | 对大规模项目效率低下 |
| **无积极维护的技术** | 安全与兼容性风险 |
| **违反 Dx 原则的技术** | 与 NLFTs 核心关注点相悖 |

> **注：** 此列表在每次月度会议上重新评估。如有充分理由，技术可从“不支持”移至“支持”，反之亦然。

## 项目与活动

### 月度会议

**会议**是 NLFTs 最独特的活动。这不是普通的聚会——它是**技术法庭**。

**形式：**

1. **提交**——提出一项技术、框架、工具或实践以供评估
2. **讨论**——所有成员深入讨论利弊
3. **裁决**——决定该技术应：
   - **采纳**——推荐在 NLFTs 生态中使用
   - **观察**——尚无定论，需进一步观察
   - **放弃**——认为低效或不相关
4. **记录**——会议结果被记录并影响 FTs Standard

**频率：** 每月一次  
**参与者：** 所有成员（贡献者和核心团队拥有优先投票权）

### 活动与研讨会

- 频率：**定期**（每月不止一次）
- 形式：线下（万隆及其他城市）+ 线上
- 范围：技术讲座、工作坊、黑客松、项目展示
- 标准：所有活动必须达到 FTs Standard 的最低质量

### 24 小时讨论

- 在 **Discord** 上进行，作为主要平台
- 主题：与开发相关的任何内容
- 没有“睡眠时间”——总有成员在线
- 高价值讨论会被归档，可作为参考

### Dx 工具与库开发

- 集体开发的工具和库项目
- 使用 FTs Standard 作为质量标准
- 使用 FTs License 作为许可证
- 通过 NLFTs GitHub 组织进行管理

## 行为准则

### 一般原则

1. **尊重其他开发者**——没有愚蠢的问题，没有侮辱性的回答
2. **专注于技术**——政治、SARA 和非技术话题在指定频道外严格禁止
3. **切实贡献**——提供解决方案比仅仅批评更好
4. **保持质量**——每条消息、PR 和文档都反映 NLFTs
5. **自立**——先自己寻找答案，但如果卡住也不要犹豫提问
6. **诚信**——不抄袭，不声称他人作品为己有

### 禁止行为

| 禁止行为 | 处罚 |
|------------|----------|
| 垃圾信息、未经授权的推广 | 警告 → 踢出 |
| 政治、SARA、仇恨言论 | **永久踢出** |
| 伪造资格/技能 | **永久踢出 + 黑名单** |
| 传播非法内容 | **永久踢出 + 报告给当局** |
| 人身攻击（而非攻击论点） | 警告 → 禁言 → 踢出 |
| 向公众泄露专属内容 | **永久踢出 + 黑名单** |
| 为个人利益滥用 NLFTs 名称 | **永久踢出 + 法律行动** |

### 违规与处罚

> 严重违规（第 4-6 级）可在**不经过先前等级**的情况下直接适用，由领导者或授权核心团队酌情决定。

## 贡献

### 如何贡献

| 贡献类型 | 方式 |
|----------------------|-----|
| **代码** | Fork 仓库 → 创建分支 → PR → 审查 → 合并 |
| **错误报告** | 使用相应模板打开问题 |
| **文档** | Fork 文档 → 修复/添加 → PR |
| **讨论** | 在 Discord 上活跃，提供高质量答案 |
| **活动** | 帮助组织、成为演讲者或引导者 |
| **审查** | 审查他人的 PR，给予建设性反馈 |

### 代码贡献标准

- 代码风格遵循 **FTs Standard**
- 对于 JS/TS 项目使用 **TypeScript**（严格模式）
- 提交信息遵循 **Conventional Commits**
- 新功能必须包含**测试**
- 如有 API 变更，必须更新文档

### 文档贡献标准

- 格式：**Markdown**（.md）
- 语言：**印度尼西亚语**（代码和技术术语可用英语）
- 语调：**专业且友好**
- 结构：遵循现有模板
- 审查：文档也通过 PR 流程

## 常见问题

### 一般

**问：NLFTs 免费吗？**  
答：是的，所有社区访问免费。无会员费。

**问：我必须住在万隆吗？**  
答：不需要。NLFTs 总部在万隆，但通过线上覆盖整个印度尼西亚。

**问：我还是学生，可以加入吗？**  
答：可以，只要您满足资格要求（扎实的基础、使用过框架、有 3 个以上项目）且年满 17 岁。

**问：为什么 PHP 和 Ruby 不被支持？**  
答：因为 NLFTs 的愿景是构建现代高效的生态系统。传统技术不符合社区核心的 Dx 关注点。此决定可在月度会议上重新审议。

### 会员资格

**问：我因不活跃被移出 WhatsApp，可以重新加入吗？**  
答：**不能。** 因不活跃被移出是永久的。不过您仍可通过 Discord 加入。

**问：如何成为 Discord 贡献者？**  
答：持续贡献——提交 PR、修复问题、在讨论中提供高质量答案。贡献者身份不是申请来的，而是**被认可的**。

**问：我是有 10 年经验的高级开发者，但没有 3 个项目可展示，可以加入吗？**  
答：项目可以是任何形式——开源、公司内部、自由职业，甚至是副业项目。只要能验证即可。

### 技术

**问：为什么关注 Nuxt 而不是 Next？**  
答：因为 Nuxt 具有出色的 Dx，目前在印度尼西亚代表性不足。NLFTs 希望填补这一空白，而不是跟随主流。

**问：我是 React/Next 开发者，我不受欢迎吗？**  
答：非常欢迎！NLFTs 对 Nuxt 的关注并不意味着我们闭门不纳其他生态。关于 React/Next 的讨论仍然允许，只是开发和推广的优先项是 Nuxt。

**问：NLFTs 会开发自己的框架吗？**  
答：目前的重点更多是在现有框架（尤其是 Nuxt）之上的**工具和库**。不过，未来可能通过会议改变。
